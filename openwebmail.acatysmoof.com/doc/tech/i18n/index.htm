Languages Overview

OpenWebMail supports Internationalization (I18N) and Localization (L10N) through the implementation of GNU gettext PO language catalogs. PO catalog files are text files that contain language strings. Translators can easily translate those strings using simple free software. They can then send the translated files back to us, giving OpenWebMail full support for their language!

Quick Start

  • Download and install the free poedit translation software
  • Choose to update an existing translation or start a new translation

Start a New Translation

  • Download the current PO template catalog file for OpenWebMail
  • Create a new catalog from the POT file in PoEdit (File->New catalog from POT file...)
  • Adjust the settings for your language:



    Plural Forms is a formula that describes how many different plurals there are in a language. English, for example, has only 2 plural forms (e.g. item, items). Arabic has 6 plural forms! In order to get all the translations for all the forms in your language you need to correctly set the plural forms formula before you begin translating. You can find the plural forms formula for your language at the Translate Toolkit website. You can read more about plural forms at the GNU website

  • Save the PO file (happens automatically after settings)
    Please use the naming convention of locale.charset.po, for example: en_US.UTF-8.po
    Your locale is a combination of your ISO 639-1 language code and your ISO 3166-1 country code
  • Translate the strings (click here for help and tips)
  • Save your work and email the PO file to the i18n mailing list or directly to the developers

Update an Existing Translation

  • Download the up-to-date PO file for your language. If your language doesn't exist start a new translation.
  • Open the PO file in PoEdit (File->Open...)
  • Translate the untranslated strings (click here for help and tips)
  • Save your work and email the PO file to the i18n mailing list or directly to the developers

Help and Tips

  • Where do I type my plurals translations?
    There is a bug in PoEdit 1.4.6 where new catalogs do not display the plural translation boxes when the catalog is first created. Simply go to Catalog->Settings... and then click ok and the plural translation boxes will appear.
  • Why do some strings look like code?
    Some strings may contain either HTML or javascript code. These strings should be more carefully translated so as not to break the code. The javascript strings almost always have single quote characters in them, for example:

    'session time: ' + (remainingseconds - 5) + ' seconds left'

    The text inbetween the single quotes (session time, seconds left) should be translated. The text that is not between single quotes (remainingseconds - 5) should not be translated. A string like this should be translated with extra care to maintain the single quotes at the start and end. If you know javascript you may change the position of the code that generates the number if your language translation requires it.

    The HTML strings almost always have a > or < character in them, for example:

    <tmpl_var availablefreespace escape=\"html\"> KB Available

    The text inbetween the carat characters should not be translated (tmpl_var availablefreespace escape=\"html\"). The text outside the carats should be translated (KB Available). If you know HTML you may move the html variable around if your language translation requires it.
  • How can I automate some translations?
    PoEdit supports automatic translations through the use of existing translations compiled into databases PoEdit calls "Translation Memories" (TM). PoEdit can reference the generated TM database and automatically translate matching strings that exist in the PO file you are translating. The use of TM is highly recommended for OpenWebMail translation because our PO files contain many country names for which translations already exist in almost all languages. Using these can be a big time saver.

    To create a TM database:

    • Designate a directory to store the translation memory databases in PoEdit (Preferences->Translation Memory tab->Browse)



    • Click the "Add" button to select your language
    • Make a directory in your TM database directory that -exactly- matches the abbreviated name of the language you just selected.
      For example: /Users/alex/Desktop/TM/it for Italian
    • Download existing ISO 3166 and ISO 639 translations for your language and put them into the abbreviated language directory you just made.
    • Click the "Generate database" button in PoEdit. You will be prompted for Search Paths. Click "Browse" and select the abbreviated language directory you made:



    • Click the "Next >" button. You should see that the PO files you downloaded previously are now listed:



    • Click the "Finish" button. The TM databases are generated


    To use the TM database you created:

    • With your PO file open, select "Catalog->Automatically translate using TM"


    PoEdit will translate all strings that have a match in the database.
  • How can I see where a string is used in the program?
    Right click any string in PoEdit to see where it is used throughout the OpenWebMail program. The context of its usage can sometimes help make a better translation.
  • Do I need to translate everything for my language?
    Absolutely not. We appreciate anything you can do! Even if you only have time to translate one string thats okay. Someone else won't have to do that work, and the project will benefit. Thank you for your contribution!
